# ![](../assets/icons/favicon-64x64.png) Aerofly Startgerät - MCP Server

The Aerofly Startgerät MCP server acts as an interface usable by local AI clients to set up flight plans as well as other flight parameters in Aerofly FS 4.

The Aerofly Startgerät MCP server is available as a pre-packacked MCPB file (which can be used with Claude Desktop), or as a stand alone Node.js application to use with any other local AI tool which can access a local MCP server.

## Installation via MCPB file

1. Be sure to have a local AI client installed which can import MCPB files.
1. Download the latest MCPB file from the Github releases.
1. Install the MCPB file by double-clicking the it.

## Installation as Node.js application

1. Be sure to have Node.js installed
1. Install the Node.js package for the Aerofly Stargerät by executing `npm install @fboes/aerofly-startgeraet`
1. Register the Aerofly Startgerät MCP server at `dist/mcp/index.js` contained in this package with your local AI client.

## Usage

After the MCP server has been registered with your local AI client, prompting the AI to build a flightplan for Aerofly FS 4 should automatically let the AI decide to use the MCP server an its capabilities.

> [!WARNING]
> Aerofly FS 4 must not be started while using the Aerofly Startgerät MCP server. Settings generated by the Aerofly Startgerät MCP server will only be read by Aerofly FS 4 on start-up of the simulator.

The MCP server offers:

- A resource with a list of all aircraft of Aerofly FS 4
- A resource with a list of all airports of Aerofly FS 4
- Tools to set the time & weather
- Tools to import live METAR weather
- Tools to set the aircraft, livery, fuel and payload
- Tools to import a SimBrief flight plan
- Tools to create a completely new flight plan
- Tools to save all of these changes to Aerofly FS 4

> [!WARNING]
> The Aerofly Startgerät MCP server may break your `main.mcf`. Be sure to have a backup of this file.
